# 更新日志 (Changelog)

## v0.10.3 (2026-06) — 预设可视化编辑

> 把游玩常用的 SillyTavern 预设（Chat Completion Preset）接进归档工具，可查看、编辑、导出。

### 🎚️ 预设编辑器（新功能）

通过顶部导航栏的「预设」入口进入。导入 SillyTavern 导出的预设 `.json`，可视化查看与编辑：

- **概览**：API 来源、模型、最大上下文 / 回复 Token、采样参数（temperature / top_p / top_k 等）、提示词块与激活条目统计
- **提示词编辑**：
  - **激活顺序**：拖拽重排、逐条启用 / 禁用、移除（带「撤销」）、展开直接编辑内容
  - **提示词库**：未加入激活顺序的条目，可一键加入；自动标记「未引用」「空条目」
  - **实时预览**：按激活顺序渲染启用条目，替换 `{{char}}` / `{{user}}`，估算 token，角色色条区分 system / user / assistant / 插槽
  - **多角色组**：预设含多个 `prompt_order` 分组时可切换；撤销 / 重做覆盖激活顺序调整
- **工具字段**：编辑 `impersonation_prompt` / `wi_format` / `scenario_format` 等工具型字段（仅显示该预设实际包含的字段）
- **正则**：编辑预设内嵌的正则脚本（`extensions.regex_scripts`），含实时测试区；复用应用统一的正则互通能力，ST 独有字段无损保留
- **导出**：完整导出（保留全部条目与未识别字段，可直接导回 SillyTavern）/ 智能导出（仅当前组启用条目）/ Markdown 文档；附「变更对比」查看相对导入时的改动
- **持久化**：导入自动保留最近 5 份历史、切换页面不丢失编辑；点「保存」永久留存（在「已存预设」中管理）；纳入完整数据备份

## v0.10.2 (2026-06) — 主编辑页存储与交互优化

> 实测大体量聊天记录切页丢失后的一轮架构与交互打磨。

### 🗄️ 跨页存储重构

- **session 本体改存 IndexedDB**：过去把整份聊天记录（可能数十万字）直接塞进 sessionStorage，而 sessionStorage 有约 5MB 硬上限——稍大的记录切页即丢、且配额按 origin 隔离，表现飘忽难复现。现改为只在 sessionStorage 存「轻量指针（当前作品 id + 章节标记 + 收藏）」，session 本体留在 IndexedDB（近乎无限），切回聊天页凭指针回读，从根上消除大记录切页丢失
- **新增「清除临时缓存」**：设置面板新增按钮，一键清空页面间的临时编辑态（不影响书架作品与世界书），切页后内容异常时可自救

### ✏️ 编辑入口改造

- **每楼铅笔直接编辑**：移除原「编辑内容模式」（需先切模式再点楼），改为每条消息右上角常驻铅笔图标——所见即点，直接打开该楼编辑窗口
- **顶栏左右分列**：外观设置与全文搜索移到顶栏左侧常驻，与右侧操作（章节标记 / 正则 / 保存 / 导入 / 导出）分列两端，弹层从左展开不再遮挡正文

### 🔍 搜索体验修复

- **搜索不再误滚**：边打字搜索词时页面不再被自动甩到第一个命中；新增正则等改动也不再触发误滚——滚动只在主动按 Enter / 上一个 / 下一个时发生

## v0.10.0 (2026-06) — SillyTavern 格式对齐

### 🌍 世界书格式加固

- **修复「禁用条目」反向 bug**：SillyTavern 用 `disable` 字段（true=禁用），本应用内部用 `enabled`。过去导入真实 ST 世界书时，被禁用的条目会被读成「启用」，导出又写错字段。现导入/导出都正确互转
- **修复导出后无法重新导入**：导出时强制让条目记录键与 `uid` 一致——这正是过去某些世界书导出后再导入会失败/丢条目的根因
- **兼容老格式世界书**：支持字符串位置（`before_char` 等）、旧字段名（`keys`/`secondary_keys`/`insertion_order` 及各 snake_case 名）、条目顶层 `name` 作标题、废弃的 `selectiveLogic:4`、`position:-1`（旧版禁用写法）
- 关键词导出为数组（对齐 ST 1.10+），过滤空/无效关键词，保留正则关键词的斜杠
- 移除并不存在的位置选项「Outlet」（ST 主流位置只有 0–6）

### 🔧 正则脚本互通

- **导入 SillyTavern 正则脚本**：正则面板新增「导入 ST」，支持单脚本、脚本数组、社区打包（`{scripts:[]}`）三种 `.json`；按脚本 id 去重合并
- **导出为 SillyTavern 正则脚本**：新增「导出 ST」，单条导出为对象、多条为数组，与 ST 双向兼容
- 自动转换应用内的「用户/AI/全部」与 ST 的数字位置枚举；ST 独有字段（trimStrings / markdownOnly / substituteRegex 等）在导入时完整保留、导出时无损还原，保留脚本 id 避免重复导入

### 🪪 角色卡查看（只读）

- **新增「角色卡」页面**：拖入 SillyTavern 角色卡（PNG 或 JSON），只读查看全部字段——描述、性格、场景、开场白、备选问候、对话示例、系统提示、标签等
- **支持 V1 / V2 / V3**：自动识别版本；PNG 同时含 V2/V3 数据时优先读 V3
- **内嵌世界书展示**：角色卡内附带的世界书条目一并解析展示（关键词、位置、常驻/禁用状态、内容预览）
- 仅查看与核对，不修改文件、不写卡（写卡功能留作后续）

### 🧹 工程

- 清理类型标注：消除 AI 工具、PNG 解析等模块的 `any`
- 测试扩充：世界书 / 正则互通 / 角色卡解析新增单元测试（总计 91 例全绿）

### ✨ 易用性 & 修复

- **删除/编辑撤销**：删除整楼、删除章节标记、编辑楼层内容（含删除楼内片段）后均弹出「撤销」按钮，一键还原；误操作不再只能重新导入

- **导出自定义范围**：楼层号同步为 0-based，按原始消息序号选取
