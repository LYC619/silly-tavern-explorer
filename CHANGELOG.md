# 更新日志 (Changelog)

## v0.10.11 (2026-06) — 角色卡页滚动修复

### 🐛 修复

- **角色卡页无法向下滚动**：移除卡片内层固定高度的滚动框（`max-h-[64vh]` 的 ScrollArea，在内容超高时滚动失效，加立绘后更易触发），改为整页自然滚动，与聊天处理 / 世界书页一致，更稳定
- **立绘加载容错**：JSON 卡的头像 URL 失效时不再显示破图标（加载失败自动隐藏立绘）

## v0.10.10 (2026-06) — 角色卡立绘显示 + 多项实测优化

### 🪪 角色卡

- **显示立绘**：导入角色卡后在卡片头部显示立绘缩略图（PNG 卡用原图，JSON 卡若带可显示的头像 URL 也会显示）；点击立绘可放大查看大图。无图的卡（多数 JSON 卡）不显示，保持简洁
- **内嵌世界书一键暂存**：角色卡内嵌世界书区新增「暂存到世界书」——把卡里的世界书存到「世界书」页（可立即前往加载/编辑/导出），免去手动搬运
- **内嵌正则一键暂存**：若角色卡内嵌正则，新增「暂存正则」——存为正则预设，到「聊天处理」页『正则 → 预设管理』即可加载用于处理聊天记录

### 🌍 世界书

- **「放大编辑」窗口加宽**：条目内容的放大编辑弹窗由原来的 768px 加宽到 1024px，长内容编辑更舒展

### 🎚️ 预设编辑器

- **导出「变更对比」改为行级高亮**：之前对比只是把内容堆在一起看不出变化；现在改为逐行红/绿高亮（绿 = 新增/改后，红 = 原值/删除），且对比时左右两侧采用相同导出模式（修复智能导出时把裁剪条目误判为大量删除的问题），真正能看清改了什么

- **术语更友好**：「插槽」徽标加悬浮说明（解释它由 ST 运行时自动填充、并非被禁用）；「工具字段」标签页更名「格式与杂项」并加入说明文字

## v0.10.9 (2026-06) — 预设支持手动新增「注入块」

### 🎚️ 预设编辑器

- **新建块支持两种类型**：「提示词」标签页的「新建块」改为下拉菜单——
  - **普通块**：按激活顺序就地插入（原有行为）
  - **注入块**：对齐 SillyTavern 的「绝对注入」（`injection_position=1`），按**深度**注入到聊天记录中，而非按激活顺序位置；新建时采用 ST 默认值（深度 4 / 顺序 100 / 触发全部生成类型）
- **注入块可编辑深度与顺序**：展开注入块时新增「注入深度 / 顺序」两个数字输入（深度 = 插入聊天倒数第几层；顺序 = 同深度的排序权重）
- 注入块在激活顺序、提示词库、实时预览三处显示专门的「注入 @深度」徽标，与普通块/插槽区分
- 导出（完整 / 智能）均完整保留注入块的 `injection_position` / `injection_depth` / `injection_order` / `injection_trigger` 字段，可直接导回 SillyTavern

## v0.10.2 (2026-06) — 主编辑页存储与交互优化

> 实测大体量聊天记录切页丢失后的一轮架构与交互打磨。

### 🗄️ 跨页存储重构

- **session 本体改存 IndexedDB**：过去把整份聊天记录（可能数十万字）直接塞进 sessionStorage，而 sessionStorage 有约 5MB 硬上限——稍大的记录切页即丢、且配额按 origin 隔离，表现飘忽难复现。现改为只在 sessionStorage 存「轻量指针（当前作品 id + 章节标记 + 收藏）」，session 本体留在 IndexedDB（近乎无限），切回聊天页凭指针回读，从根上消除大记录切页丢失
- **新增「清除临时缓存」**：设置面板新增按钮，一键清空页面间的临时编辑态（不影响书架作品与世界书），切页后内容异常时可自救

### ✏️ 编辑入口改造

- **每楼铅笔直接编辑**：移除原「编辑内容模式」（需先切模式再点楼），改为每条消息右上角常驻铅笔图标——所见即点，直接打开该楼编辑窗口
- **顶栏左右分列**：外观设置与全文搜索移到顶栏左侧常驻，与右侧操作（章节标记 / 正则 / 保存 / 导入 / 导出）分列两端，弹层从左展开不再遮挡正文

### 🔍 搜索体验修复

- **搜索不再误滚**：边打字搜索词时页面不再被自动甩到第一个命中；新增正则等改动也不再触发误滚——滚动只在主动按 Enter / 上一个 / 下一个时发生

## v0.10.0 (2026-06) — SillyTavern 格式对齐

### 🌍 世界书格式加固

- **修复「禁用条目」反向 bug**：SillyTavern 用 `disable` 字段（true=禁用），本应用内部用 `enabled`。过去导入真实 ST 世界书时，被禁用的条目会被读成「启用」，导出又写错字段。现导入/导出都正确互转
- **修复导出后无法重新导入**：导出时强制让条目记录键与 `uid` 一致——这正是过去某些世界书导出后再导入会失败/丢条目的根因
- **兼容老格式世界书**：支持字符串位置（`before_char` 等）、旧字段名（`keys`/`secondary_keys`/`insertion_order` 及各 snake_case 名）、条目顶层 `name` 作标题、废弃的 `selectiveLogic:4`、`position:-1`（旧版禁用写法）
- 关键词导出为数组（对齐 ST 1.10+），过滤空/无效关键词，保留正则关键词的斜杠
- 移除并不存在的位置选项「Outlet」（ST 主流位置只有 0–6）

### 🔧 正则脚本互通

- **导入 SillyTavern 正则脚本**：正则面板新增「导入 ST」，支持单脚本、脚本数组、社区打包（`{scripts:[]}`）三种 `.json`；按脚本 id 去重合并
- **导出为 SillyTavern 正则脚本**：新增「导出 ST」，单条导出为对象、多条为数组，与 ST 双向兼容
- 自动转换应用内的「用户/AI/全部」与 ST 的数字位置枚举；ST 独有字段（trimStrings / markdownOnly / substituteRegex 等）在导入时完整保留、导出时无损还原，保留脚本 id 避免重复导入

### 🪪 角色卡查看（只读）

- **新增「角色卡」页面**：拖入 SillyTavern 角色卡（PNG 或 JSON），只读查看全部字段——描述、性格、场景、开场白、备选问候、对话示例、系统提示、标签等
- **支持 V1 / V2 / V3**：自动识别版本；PNG 同时含 V2/V3 数据时优先读 V3
- **内嵌世界书展示**：角色卡内附带的世界书条目一并解析展示（关键词、位置、常驻/禁用状态、内容预览）
- 仅查看与核对，不修改文件、不写卡（写卡功能留作后续）

### 🧹 工程

- 清理类型标注：消除 AI 工具、PNG 解析等模块的 `any`
- 测试扩充：世界书 / 正则互通 / 角色卡解析新增单元测试（总计 91 例全绿）

## v0.9.2 (2026-06)

### 🚀 性能

- **长聊天记录虚拟化**：阅读区只渲染屏幕可见的消息（外加少量预备行），无论 464 楼还是更长，首屏与滚动都流畅；过去全量渲染上万节点导致的卡顿消除
- **路由懒加载**：各页面按需加载，首屏不再下载整个应用；打包从 659KB 单文件拆成主包约 290KB + 各页独立分片
- **依赖精简**：移除未使用的 react-query / zod / date-fns

### 🛡️ 数据安全

- **完整备份纳入世界书**：原「完整备份」只导出聊天作品、漏掉世界书，清浏览器数据会静默丢失；现备份/恢复同时覆盖作品与世界书（兼容旧备份文件）
- **临时缓存溢出提示**：超大记录暂存超出浏览器上限时不再静默失败，改为提示「请保存到书架」，避免跨页丢失未保存编辑

### 🔖 跳楼层 & 收藏

- **左侧悬浮跳转条**：固定在屏幕左侧（不随滚动消失、不挤占正文），支持输入楼层号跳转、上/下一层、收藏当前楼层、收藏列表一键跳转
- **收藏（轻量书签）**：独立于章节标记、不进导出，跟随作品持久化保存
- **常驻楼层号**：每条消息左侧显示楼层号，方便对照
