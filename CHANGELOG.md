# 更新日志 (Changelog)

## v0.9.1 (2026-06)

### 🏗️ 界面重构

- **左侧主导航栏**：新增统一的 `AppLayout`，聊天处理 / 世界书 / 书架 / AI 工具四大功能以图标导航，当前页高亮；移除各页面重复的顶部导航与"绕回首页才能切换"的星型跳转
- **工具栏合并单行**：原先分占两行的操作合并为一行，按「外观 ┃ 编辑·章节·正则 ┃ 保存·导入·导出」逻辑分组；外观设置（主题/字号/阅读宽度/时间戳/字体/导出格式）收进 popover，为阅读区腾出空间；导出为唯一主操作并高亮
- **正则预览主界面化**：取消正则侧边栏的小框对比，改为在阅读区原地以红色删除线 / 绿色替换高亮标出规则效果，所见即所得
- **起点式阅读排版**：阅读区按段落渲染，每段独立首行缩进、舒适行距；同一说话人连续发言省略重复名字
- 载入聊天记录后自动展开正则规则栏

### 🐛 问题修复

- **世界书白屏**：修复因回调声明顺序导致的渲染崩溃（TDZ）
- **设置按钮无法点击**：新手引导找不到目标时不再用全屏遮罩阻塞页面
- **书架/阅读器正则**：阅读器改用作品保存时的正则规则（原先因 localStorage 键名错误总是回退到默认规则）
- **导出文件大小计算**：改用真实 UTF-8 字节数（中文每字约 3 字节），原始 / 导出口径统一
- **消息引号**：去除预览中每条消息被自动包裹的装饰性引号
- **章节标记错位**：导出 / 阅读器 / 预览统一按消息 ID 匹配章节标记，删除或重排消息后不再跑偏
- **章节标记按钮重叠正文**：标记模式下按钮移入消息顶部留白区
- **时间戳不显示**：补齐 elegant/novel 主题的时间戳渲染；正确解析 SillyTavern 的两种日期格式（`November 14, 2024 6:18am` / `2024-11-14 @06h 18m 30s 500ms`），旧数据自动兜底解析

### ⚙️ 正则与体验

- 清空过时的内置正则规则，改由「快速添加」自建
- 快速添加新增「内容替换」模式（通篇将 A 替换为 B）
- 正则预览扫描全部消息（不再只看开头若干条而误报"无匹配"）
- 手动添加正则后自动滚动到新规则
- 大文本性能优化：正则编译实例缓存 + 预览处理防抖

## v0.9.0 (2026-03)

### 🎓 完整新手引导系统

- **分步引导 (Guided Tour)**：首页、书架、世界书、AI 工具四个模块各自拥有独立的分步引导，高亮目标元素逐步讲解
- **自动触发**：首次访问自动加载示例数据并启动引导，完成后可在设置中重置
- **引导状态持久化**：各模块引导完成状态独立保存到 localStorage

### 📚 书架优化

- **排序切换**：书架页面支持按更新时间、创建时间、标题字母排序
- **渐变占位封面**：无封面书籍自动生成基于书名哈希的渐变色块 + 居中书名文字，支持自适应字号与文字阴影

### 🔤 字体选择

- **阅读器字体**：沉浸阅读模式新增字体选择（系统默认/宋体/楷体/黑体/等宽），选择保存到 localStorage
- **预览字体**：首页设置面板「更多设置」中新增预览字体下拉，应用到聊天预览区域
- **外部字体引入**：通过 Google Fonts 引入 LXGW WenKai 和 Noto Sans SC

## v0.8.0 (2026-03)

### 🌍 世界书可视化编辑器（新功能）

- **导入/导出**：支持导入 SillyTavern 世界书 JSON（兼容记录格式与数组格式），导出为标准 JSON
- **追加导入**：支持将新世界书追加到已有数据，自动处理 UID 冲突
- **卡片视图 & 列表视图**：双视图切换，卡片展示标题、策略标识（🔵常驻/🟢关键词/🔗向量）、关键词标签及基于 Inclusion Group 的 HSL 颜色条
- **编辑面板**：完整的条目编辑器，支持原子化策略切换、逻辑模式、递归设置、深度与角色配置；移动端适配底部抽屉式编辑器
- **搜索筛选排序**：支持按标题/关键词/内容搜索，按策略和启用状态多维筛选，按 Order/标题/UID 排序
- **快速创作模式**：粘贴空行分隔文本自动拆分为条目，启发式提取标题和关键词，支持全局默认值配置
- **前缀归类**：扫描条目标签（group 字段），为未分类条目批量分配标签，自动添加「标签--标题」前缀并按标签分组重新编号 Order

### 🎓 新手引导（基础版）

- 三步式快速上手卡片：导入 → 处理 → 导出
- 「不再显示」复选框，状态持久化到 localStorage

### 🐛 v0.7 遗留修复

- 修复 `@radix-ui/react-toggle` 和 `@radix-ui/react-toggle-group` 依赖缺失导致 Vercel 构建失败
- 移除未使用的 `accordion.tsx` 组件
- 清理 `jsdom` 重复依赖（从 dependencies 移至仅 devDependencies）

### 🔧 构建修复

- 修复构建时因缺失 Radix UI 包导致的编译错误
- 确保所有 UI 组件依赖在 package.json 中正确声明
