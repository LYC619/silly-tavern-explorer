## v0.16.1 (2026-07) — 分栏与弹窗显示顽疾根治（去断点化）

### 🐛 修复（同类问题第四次出现，这次移除依赖而非调参数）

- **总结页 / 故事树页上下堆叠（Edge、Chrome 均复现）**：排查确认构建 CSS 完好（`sm:` 媒体查询存在且顺序正确），而全程用普通 flex 布局的世界书页从未出过问题——问题出在「视口断点」这条路径在用户环境（高缩放 × 高分屏 × 注入式插件）下反复失灵，断点 1024→768→640 一路调低仍会踩空。现将 总结工作台(5:7)、展示页(4:8)、故事树(2:3) 三处分栏全部改为 **flex-wrap + 行内 flex-basis**：容器放得下两栏就分栏、放不下自动换行成单列，与视口宽度、媒体查询、Tailwind 类生成/清除、插件类覆盖完全解耦（行内样式随 JS 发布，不依赖 CSS 规则存活）
- **世界书「放大编辑」弹窗在 Edge 缩成小窗**：实测该环境下 `!max-w-[min(110rem,94vw)]` 类规则不生效，弹窗回落到 shadcn 底座的 `max-w-lg`(512px)；宽度改加**行内样式兜底**（`maxWidth: min(110rem, 94vw)`），不依赖类规则存活，稳压底座默认值

## v0.16.0 (2026-07) — 隐藏楼层导入修复 + 总结页展示/书签/卷号治理 + 故事树分卷分段

### 🐛 修复

- **导入丢失被 Hide 的楼层**：ST 的「Hide message」是把 `is_system` 置 true 持久化的，此前导入器把 `is_system` 一刀切跳过，被隐藏的真实楼层（如开场白）永远进不来。现在只跳过真·系统提示（mes 为空 / 无 name 且无 is_user 的纯注入 / `extra.type` 为 narrator·system），其余照常导入并打 `hidden` 标记；聊天页有隐藏楼层时顶部出「显示/隐藏 N 个隐藏楼层」开关（默认展示、楼内标「ST 中已隐藏」角标）；导出 JSONL 时按当前 hidden 状态写回 `is_system`，round-trip 与 ST 行为一致。**此前导入的书需重新导入一次才能找回被丢弃的隐藏楼层**
- **起止楼层联动乱跳（总结页 + 故事树 AI 弹窗同源）**：楼层选择器此前在输入时做反向夹取（改结束楼层会 `Math.min` 连带改起始楼层），且空串被 parse 成 0——清空重填时另一端瞬间被拽到 0。现两输入框各自独立、允许输入中途为空，只在失焦时规范化（空串回填、起>止对调）
- **总结页切页后楼层回退**：楼层区间/类型/日记主角现记入跨页轻量态（sessionStorage，按书隔离），去聊天页确认范围再回来不再被重置为默认全书
- **分卷卷号乱跳（生成 +1、保存又 +1）**：卷号改由「实际情况」推断（`inferVolumeNumber`）——起始楼层与已有某卷一致视为重做该卷、沿用其卷号，否则 = 最大卷号 + 1；且生成时盖章卷号，保存沿用盖章值，不再受生成后自动落库引起的 priors 变化影响

### 📖 总结页

- **展示页**：新增与「生成工作台」并列的顶层视图，生成与阅读分离——左侧按 当前书/类型 筛选条目（分卷按卷号排序），右侧纸张感衬线排版阅读，日记走日记本渲染；支持复制 / 下载 .md / 一键「去编辑」回工作台
- **手动添加总结**：右栏新增「手动添加总结」按钮，展开空白编辑器直接书写，保存即入库（不经 AI），与生成结果同一套编辑/导出流
- **书签楼层**：楼层选择器新增「书签楼层」弹层，列出聊天页的章节标记与收藏楼层，一键填入起点/终点
- **日记主角**：日记 tab 新增「生成谁的日记」填空，非空时自动在提示词末尾附加「以「XX」的第一人称视角生成 XX 的日记」，随 genParams 记忆、重新生成可回填

### 🌳 故事树

- **累加式生成说明**：AI 生成弹窗标明当前树结构（含之前各卷节点）会随请求发给 AI、AI 只输出增量更新——按第二卷生成无需重带第一卷
- **条目正文按卷分段**：AI 应用新事实时归入 `## 第N卷 · 楼层 a~b` 小节（选中范围落在已存分卷内用卷号，否则用楼层区间；同卷重复生成并入既有小节不重复建头）；节点编辑器对含小节的正文默认提供「分卷」只读视图，一眼看清角色跨卷状态变化，点「编辑」回纯文本

### 📄 聊天记录页

- **标题就地重命名**：预览区顶部信息行显示作品标题，点击铅笔就地改名，随书架自动同步、导出文件名同步生效

## v0.10.2 (2026-06) — 主编辑页存储与交互优化

> 实测大体量聊天记录切页丢失后的一轮架构与交互打磨。

### 🗄️ 跨页存储重构

- **session 本体改存 IndexedDB**：过去把整份聊天记录（可能数十万字）直接塞进 sessionStorage，而 sessionStorage 有约 5MB 硬上限——稍大的记录切页即丢、且配额按 origin 隔离，表现飘忽难复现。现改为只在 sessionStorage 存「轻量指针（当前作品 id + 章节标记 + 收藏）」，session 本体留在 IndexedDB（近乎无限），切回聊天页凭指针回读，从根上消除大记录切页丢失
- **新增「清除临时缓存」**：设置面板新增按钮，一键清空页面间的临时编辑态（不影响书架作品与世界书），切页后内容异常时可自救

### ✏️ 编辑入口改造

- **每楼铅笔直接编辑**：移除原「编辑内容模式」（需先切模式再点楼），改为每条消息右上角常驻铅笔图标——所见即点，直接打开该楼编辑窗口

### 🔍 搜索体验修复
